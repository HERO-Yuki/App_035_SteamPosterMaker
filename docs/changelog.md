## v17（2026-04-10）— コードレビュー・リファクタリング（第2弾）

- `_CACHE_TTL / _CACHE_MAX_SEARCH / _CACHE_MAX_DETAILS / _CACHE_MAX_IMAGES` を新設

## v16（2026-04-10）— Streamlit Cloud メモリ使用量の最適化

**@st.cache_data への max_entries 追加**
- `search_steam`: max_entries=100
- `get_game_details`: max_entries=200
- `_fetch_raw_image`: max_entries=50（画像バイト列はサイズが大きいため最小値）

**Pillow オブジェクトの明示的解放**
- `poster.save(buf, ...)` 完了後に `poster.close()` を追加

**セッション状態の先行クリーンアップ**
- 生成ボタン押下直後、`st.status` ブロック開始前に `last_poster_bytes` / `last_poster_meta` を `pop`
- 新旧ポスター画像（各 〜5 MB）がメモリ上に同時存在する瞬間を排除

## v13（2026-04-10）— プレイ人数フィールド完全撤廃

**背景・目的**
- プレイ人数行（`PLAYER_H = 26px`）がポスター上のレビューエリアを圧迫し、紹介文の記述量が制限されていた
- 入力したければユーザーがレビュー文に自由記述できるため、専用フィールドは不要と判断

**app.py 変更内容**
- 定数 `PLAYER_H`, `PLAYER_FONT_PT`, `PLAYER_MIN_PT`, `PLAYER_PRESETS` を削除
- `compute_layout`: `player_y` 計算・戻り値を削除。`review_y = title_y + title_max_h + ROW_GAP` に簡略化
- `draw_card`: プレイ人数描画ブロック撤廃。プレイ人数空き時の `freed` 補正ロジックも不要になり削除。レビュー描画を常に `L["review_y"] / L["review_max_h"]` を使う1パスに統一
- `edit_dialog`: `st.multiselect("プレイ人数")` を削除。文字数上限を `int(_L["review_max_h"] * 140 / 84)` で計算（84px = 旧基準高さ）。`text_area` height を 120→160px に拡大
- `render_slot_card`: プレイ人数行の表示を削除
- ゲームデータ初期化・保存・クリア処理から `players` キー・`dlg_players_{i}` session_state を全て除去
- `_DEV_SAMPLE_GAMES` から `"players"` フィールドを削除

**効果**
- review_max_h が約 38% 増加（10-game, show_title=True: 84px → 116px）
- max_chars 上限が 140 → 約 193 文字（同条件）に自動拡大
- 8-game モードや no-title モードではさらに増加

## v11（2026-04-10）— テーマカラー変更・セール強調・ボタン整理

**テーマカラー変更（config.toml）**
- `primaryColor: "#66c0f4"` → **`"#A4D007"`**（Steam セールグリーン）
- `type="primary"` が指定されている「ポスターを生成」ボタン・進捗バーが自動的にグリーンになり、ダークネイビー背景に対して際立つ視線誘導を実現

    - 定価部分 (`¥YYY`) → `theme["accent"]` で描画

## v10（2026-04-10）— ポスターフッター帯・並び替え UX 改善・UI 細部修正

**ポスター: フッター帯の導入**
- `FOOTER_H = 36`（px）定数を新設。`compute_layout` で `grid_h` を `CANVAS_H - header_h - FOOTER_H` に変更し、カードグリッド下端にフッター帯の余白を確保
- `generate_poster` でカード描画後にフッター帯（`theme["header"]` 色 + `theme["accent"]` 4px ライン）を描画
- ウォーターマーク（"Generated by SteamPosterMaker"）をフッター帯内で `textbbox` を使い縦中央揃えに配置。8枚目カードと重なっていた問題を解消
- `HEADER_H: 120 → 96 px`（20%削減）。フォントサイズ(64pt)はそのまま収まるサイズ。ヘッダーとフッターが対称デザインになり、ポスター全体の統一感が向上

**スロット並び替え: 誤操作防止**
- `render_slot_card(i, disabled=False)` に `disabled` パラメータを追加
- 並び替えモード時（`reorder_mode=True`）、編集ボタンに `disabled=True` を渡してホバーエフェクトを抑制・クリック無効化
- カードグリッド描画を `if/else` から **常時描画** に変更: 並び替えモードでも `sort_items` リストの下にカードグリッドを表示し、無効化された編集ボタン付きでゲーム情報を参照可能に

**免責事項: HTML リンクの修正**
- `st.expander("利用規約・免責事項")` 内の `st.markdown` に `unsafe_allow_html=True` を追加
- 「Streamlit」「利用規約」の `<a>` タグがリンクとして機能するよう修正（従来はタグが文字列のまま表示されていた）

**X ボタン: ホバーエフェクト追加**
- `_X_BUTTON_HTML` に `<style>` ブロックを埋め込み、`.x-btn` クラスで CSS `transition` と `:hover` を定義
- ホバー時: 背景 `#000 → #1a1a1a`、ボーダー `#333 → #666`、`opacity: 0.85`（0.2s ease）

## v8（2026-04-10）— ダイアログ UI 大幅刷新

**ダイアログ 2フェーズ化**
- `edit_dialog` を「検索フェーズ」と「編集フェーズ」に完全分離

    - `render_slot_card` と `edit_dialog` の両方で使用していた同一の囲み枠バッジ HTML を統一
