# SteamPosterMaker — デザイン・仕様決定ログ

開発中に議論・決定した UI/UX 仕様をまとめた記録です。  
「なぜそうなったか」を後から追えるようにしています。

---

## 1. レイアウト

### Q: グリッドは横4×縦2 と 横2×縦4 のどちら？

**決定: 横2×縦4（1行に2タイトル、4段）**

| 比較 | 横4×縦2 | **横2×縦4（採用）** |
|------|---------|----------------|
| 1カードの横幅 | 狭い（約460px） | 広い（約950px） |
| タイトルの読みやすさ | 長いタイトルが詰まる | 余裕がある |
| サムネイル | 小さい | 大きく表示できる |

Twitter向け横長 1920×1080 に「縦4段」が最も見やすい。

---

### Q: 全体見出し ON/OFF で何本紹介できる？

**決定: ON → 8本（ヘッダー 120px + 2×4）／ OFF → 10本（2×5）**

- 画像サイズは常に 1920×1080 px で固定
- ヘッダーなしにすると 5段になり 10本紹介できる
- toggle `st.toggle("全体見出しを表示する")` で切り替え

---

### Q: カード間のマージンは？

**決定: 10px（当初 20px から変更）**

- 20px では余白が目立ち、カードが小さく見えた
- 10px に変更 → カードサイズが約 950×227 px（8本モード）に拡大

## 3. 背景スタイル

**決定: 2択（ぼかし / 単色）**

- **ぼかし**: ゲームのヘッダー画像を拡大・ガウスぼかし（強度スライダー 1〜40）
- **単色**: テーマカラーの背景色を使用
- 年齢制限ゲームのカードはぼかし処理をスキップ（錠前画像を使うため）

## 4. 価格バッジ

### Q: 価格の表示位置は？

**決定: サムネイル右下・半透明バッジ（サムネ端から 10px 余白）**

- 当初はテキストエリア内に表示していた
- サムネ右下バッジに変更 → テキストエリアの高さが増え、レビュー文が多く書ける
- バッジ: 黒半透明背景 + アクセントカラー文字

### Q: バッジのサイズ感は？

**決定: フォント 24pt・端から 10px の余白**

- 当初フォント 16pt、端ビタ付け
- 16pt は小さすぎて読みにくいと判断 → 1.5倍の 24pt に変更
- 「ビタ付け」が気になるとのフィードバック → `EDGE_MARGIN = 10px` を追加

---

## 5. 年齢制限コンテンツ

### Q: Steam API が年齢制限で詳細を返せない場合の表示は？

**決定: テキスト「エラー」ではなく、Pillow で描画した錠前アイコン + "18+"**

- カード背景: ダークレッド
- 錠前ボディ（角丸矩形）+ アーチ（楕円×2）+ キーホール（円+三角）を Pillow で描画
- UI プレビューカードには `🔞` 絵文字を表示
- `@lru_cache(maxsize=4)` で同サイズの錠前画像をキャッシュ

---

## 6. レビュー文

### Q: 文字数上限をどう扱う？

**決定: 入力は自由、140字超で保存不可（リアルタイムカウンター付き）**

| 案 | 内容 |
|----|------|
| `max_chars=140` で強制打ち切り | 絵文字を含む長い文が途中でカットされる違和感 |
| **リアルタイムカウンター + 保存無効化（採用）** | 何文字入力したかが見え、超過時に「減らしてから保存」と案内できる |

- 140字超: カウンター文字色が赤、`st.error` 表示、「💾 保存して閉じる」が `disabled=True`

### Q: ポスター上のフォントで実際に何文字収まる？

- 現在のフォントで等幅換算すると約 148文字相当が4行に収まる見込み
- ただしゲーム名・価格・プレイ人数との兼ね合いで **140文字を上限として維持**
- ポスター上では収まらない場合、フォントサイズを 1pt ずつ自動縮小

---

## 7. 全体見出し（ポスタータイトル）

### Q: 文字数上限は？

**決定: 25文字**

- ヘッダー幅 1920px、フォント 64pt、日本語全角 1文字 ≈ 70px
- 左右パディング 80px を引くと `(1920 - 80) / 70 ≈ 26.3` 文字
- 日本語タイトルを想定して安全マージンを取り **25文字** に設定
- UI 側は `max_chars=25` で強制

---

## 8. スロット並び替え（ドラッグ＆ドロップ）

### Q: ドラッグ＆ドロップが効かない（不具合）

**原因**: `sort_items()` を `st.expander` 内に置いていたため、ドラッグ後の rerun でエキスパンダーが閉じ、ユーザーに「変化なし」に見えた。

**決定: 並び替えモードのトグルを導入**

- スロットヘッダー行に **「🔀 並び替え」ボタン** を追加
- 押すと 2×N グリッドが**縦1列のソータブルリスト**に切り替わる
- 「✅ 並び替え完了」ボタンで通常グリッドに戻る
- ドラッグ領域が明確になり、誤操作も防止

---

## 9. インポート機能

### Q: 「最近プレイしたゲーム」「プレイ時間 TOP」を API 経由で自動取得できる？

**当初実装 → 削除**

- Steam Web API の `IPlayerService` は **API キーが必要**
- ユーザーへの API キー発行・入力の負担が大きいと判断
- **ウィッシュリスト取得（API キー不要）のみ残した**

| 機能 | 必要なもの | 採用 |
|------|-----------|------|
| ウィッシュリスト | なし（公開設定のみ） | ✅ 採用 |
| 最近プレイしたゲーム | Steam API キー | ❌ 削除 |
| プレイ時間 TOP | Steam API キー | ❌ 削除 |

## 12. アプリ名変更・定数集約（2026-04-10 v5）

### Q: アプリ名は「Steam8 Poster」のままでよいか？

**決定: `SteamPosterMaker` に変更**

- "8" の数字が10本対応後に誤解を生む（8本専用アプリに見える）
- 動詞 "Maker" を付けることで「作るツール」の意図が伝わりやすい
- 変更箇所: `APP_NAME` 定数・`st.title`・`page_title`・ファイルコメント（1か所変えれば他は自動追従）

### Q: カード間マージンをどこまで狭められるか？

**決定: `MARGIN = 4 px`**

| 値 | カードサイズ（見出しON） | 印象 |
|----|------------------------|------|
| 20 px（初期） | 930×215 px | 余白が広く窮屈 |
| 10 px | 950×227 px | バランスが良い |
| **4 px（採用）** | **954×235 px** | 余白最小・コンテンツ最大化 |

4px は「隣のカードと完全に接しない最小の視覚的区切り」として残す。0にすると境界が消えて読みにくくなるため採用しない。

### Q: ハードコードされた数値が `compute_layout` や `draw_card` に散在していた

**決定: `app.py` 冒頭の定数セクションに集約**

- グリッド系: `HEADER_H`, `PLAYER_H`, `ROW_GAP`, `TITLE_MAX_H_ON/OFF`
- タイポグラフィ系: `HEADER_FONT_PT`, `TITLE_FONT_PT`, `TITLE_MIN_PT`, `PLAYER_FONT_PT`, `PLAYER_MIN_PT`, `REVIEW_FONT_PT`, `REVIEW_MIN_PT`, `PRICE_FONT_PT`, `SLOT_PH_FONT_PT`, `WM_FONT_PT`
- 既存: `MARGIN`, `THUMB_W`, `SEPARATOR_W`, `TEXT_PAD`, `PRICE_BADGE_PAD`, `PRICE_BADGE_EDGE`

これにより「どの数値を変えればどこに影響するか」がファイル冒頭だけで把握できる。

## 13. UX 改善バッチ（2026-04-10 v6）

### Q: フォントダウンロードが GitHub 障害時に完全失敗する

**決定: `FONT_URLS` リストで複数 URL を順番に試みる**

- プライマリ: GitHub 公式リポジトリ
- フォールバック: jsDelivr CDN（GitHub 障害時に代替）
- 全 URL 失敗時は Pillow デフォルトフォントにグレースフルデグラデーション

### Q: ダウンロードファイル名にタイトルを含めたい

**決定: `_safe_filename()` ヘルパーを追加**

- ポスタータイトルから使用不可文字（`/ \ : * ? " < > |`）を `_` に置換
- 最大20文字に切り詰め
- ファイル名例: `steam_8pick_2026年_神ゲー8選_20260410.png`
- 見出しOFF（10本モード）の場合はタイトルなし: `steam_10pick_20260410.png`

### Q: 見出しトグルOFF時に無効化入力欄を表示したままにするべきか？

**決定: トグルOFF時は入力欄を非表示にする**

- `disabled=True` の入力欄はユーザーを混乱させる（灰色で入力できない）
- OFF 時は `st.caption("見出しなし — 10本紹介モード")` のみ表示
- session_state の値（"poster_title" キー）はウィジェット非表示時も保持されるため、ON に戻すと入力内容が復元される

### Q: UIプレビューカードのレビュー文に改行が反映されない

**決定: `st.caption` → HTML `<p>` + `<br>` に変更**

- `st.caption` は改行を無視する
- XSSエスケープ（`&`, `<`, `>` の変換）後に `\n` → `<br>` に変換して `st.markdown(..., unsafe_allow_html=True)` で描画

### Q: テーマ選択時に実際の色が見えない

**決定: セレクトボックスの直下にカラースウォッチ（3色）を表示**

- 表示色: 背景（`bg`）・アクセント（`accent`）・カード（`card_bg`）
- 22×22 px の丸角スパンをインライン HTML で描画（`st.markdown(unsafe_allow_html=True)`）
- テーマ変更のたびにポップオーバー内で即座に更新

### Q: AppID がわかっているゲームをキーワード検索なしで直接設定したい

**決定: 検索入力欄で数字のみ入力した場合は AppID として直接取得**

- `q.isdigit()` で判定（5桁未満も含む全数字文字列を対象）
- 検索 → 候補選択 → 確定の3ステップをスキップし、`get_game_details(app_id)` を直接呼び出す
- 年齢制限・データ取得失敗時は警告を表示してスロットにフォールバック値を設定
- プレースホルダ文字を "タイトル（日英）または AppID（例: 570）を入力して Enter" に更新

## 14. ビジュアル・UX 改善バッチ（2026-04-10 v7）

### Q: ポスターのサムネイルが上下や左右で切れてしまう

**決定: cover（拡大＋中央クロップ）→ contain（縮小＋letterbox）に変更**

- Steam ヘッダー画像のアスペクト比は約 2.14:1（460×215）
- サムネ枠は 380×235（比率約 1.62:1）→ 横方向が相対的に短く、上下がクロップされていた
- `min()` スケールで縮小・中央揃えで黒背景に貼り付ける `load_pil_image_contain()` を新規追加
- 背景ぼかし用の `load_pil_image()` は cover のまま維持（フル面積で埋める用途に適切）

### Q: ダイアログで選択したゲームが本当にあってるか確信が持てない

**決定: Progressive Disclosure 展開後にヘッダー画像を width=250 でプレビュー表示**

- 検索結果一覧でも小サムネを出しているが、確定後の「編集モード」でも確認できるよう追加
- `width=250` 固定：ダイアログ幅（large）は約 700px、隣にタイトル・価格テキストを並べる 5:7 カラム構成
- 年齢制限ゲームは画像取得不可のため `st.warning("🔞 画像を取得できません")` でフォールバック
- レビュー文・プレイ人数入力をフルwidthに変更し、入力スペースを拡大

### Q: UIカードの価格テキストが他の情報と区別しにくい

**決定: Steam アクセントカラー（#66c0f4）の囲み枠バッジで表示**

- `border:1px solid #66c0f4` + `border-radius:4px` の `<span>` で実装
- ポスター上の半透明バッジと意味的に対応させたデザイン
- `white-space:nowrap` を指定して改行を防止（「-40% ¥ 300」形式は空白含む）

### Q: プレイ人数の表示ラベルがなく何の情報かわかりにくい

**決定: 「プレイ人数: ソロ / オンライン協力」形式で表示**

- ラベルを明示することで、タイトルや価格との視覚的区別がつきやすい
- スロットカードの `players_line = f"プレイ人数: {players_raw}"` で実装

### Q: 免責事項・利用規約が少なく、友人に渡す際に不安

**決定: 5セクション構成に拡充・X ボタンをブランドカラー化**

- 追加セクション: 「ユーザーの入力内容に関する責任」「データの保持について」
- Steam API の年齢制限による画像取得不可を【重要】付きで明示
- X リンクを文字リンク → ブランドカラーボタン（黒背景・白文字・SVGロゴ）に変更

### Q: グローバル CSS とリンクボタン HTML が main() 内に散在していた

**決定: モジュール定数（`_GLOBAL_CSS`, `_X_BUTTON_HTML`）に抽出**

- ファイル冒頭の定数セクションに配置し `main()` は定数名で参照
- HTML/CSS の変更箇所が明確になり、ロジックコードとの混在が解消

## 15. ダイアログ UI 大幅刷新（2026-04-10 v8）

### Q: 検索フォーム・候補一覧・ゲーム画像・入力フォームが縦に積み重なり、ダイアログが煩雑だった

**決定: 「検索フェーズ」と「編集フェーズ」を状態フラグで完全分離**

| フェーズ | 表示内容 | 遷移条件 |
|----------|----------|----------|
| 検索フェーズ | 検索フォーム + 候補リスト（＋サムネプレビュー） | ゲーム未選択 または `dlg_search_back_{i}` フラグ存在時 |
| 編集フェーズ | 「検索に戻る」ボタン + 2カラムUI + アクションボタン | ゲーム選択済み かつ フラグなし |

- 状態管理は `session_state[f"dlg_search_back_{i}"]` フラグ1本で行う（True/False ではなく「存在/不存在」）
- フラグは「保存・クリア・キャンセル」でダイアログを閉じる際に必ず `pop()` してクリアする（残留バグを防ぐ）

### Q: 検索候補画像と選択済みゲーム画像が縦に2枚並んでいた

**決定: 編集フェーズで検索フォームと候補リストを非表示にする + 2カラムレイアウト**

- `st.columns([1, 2])` で左=ゲーム画像（比率1）/ 右=タイトル・価格・プレイ人数・レビュー（比率2）
- 画像は変わらず `_show_age_restricted_thumb()` フォールバックに対応
- 右カラムにタイトル・価格バッジを配置することで「ゲームを確認しながら入力する」UX を実現

### Q: `_price_badge_html` の同一 HTML が `render_slot_card` と `edit_dialog` の2箇所に存在していた

**決定: `_price_badge_html(price_raw: str) -> str` として `_show_age_restricted_thumb` の直下に抽出**

- XSS 対策の `html.escape()` を一元化（以前は片方だけ適用していた可能性があった）
- 定数セクション直後のヘルパー関数として配置し、モジュール全体から参照しやすくする

## 16. サイドバー強化・開発者モード（2026-04-10 v9）

### Q: 登録したゲームが何本か、スクロールしないとわからない

**決定: サイドバー最上部に `st.progress` で進捗バーを表示**

- `filled / num_games` を割合で渡し、テキストに「進捗: 4 / 8 本」を付与
- `num_games` はトグル状態に応じて 8 or 10 が動的に反映される
- サイドバーは折りたたみ可能なため、常時確認したいユーザーはサイドバーを開いたままにできる

### Q: 生成ボタンがページ下部にあり、ゲームを入力するたびにスクロールが必要

**決定: サイドバーに CTA ボタンを常駐配置**

- `key="sidebar_generate_btn"` で主ボタン（`generate_btn`）と区別
- `generate_btn or sidebar_generate_btn` でどちらから押しても同じ生成ロジックを実行
- `filled == 0` 時は `disabled=True`、生成済みは「再生成」に切り替わる（主ボタンと完全同期）

### Q: 毎回 Steam API を呼んでテストするのが手間

**決定: `DEV_MODE` フラグ + `_DEV_SAMPLE_GAMES` によるテストデータ一括入力**

- `DEV_MODE: bool = True` を定数セクションに配置。`False` に変えるだけで非表示になる
- `_DEV_SAMPLE_GAMES` は API を呼ばず直接 session_state に書き込む。ネット不要でローカル開発を高速化
- `random.sample` で毎回異なる組み合わせが入るため、レイアウト確認に便利
- サンプルは12タイトル（8本・10本モードどちらでも重複なしで対応できる最小数）

### Q: 先行計算とウィジェット宣言の順序が混在していた

**決定: `show_title / layout / num_games / filled / already_generated` をサイドバー前で一括先行計算**

- サイドバー内の進捗バーとCTAボタンが `filled` / `already_generated` を必要とするため、
  `with st.sidebar:` の前にまとめて計算する構造に整理
- 同一リラン内でトグルが値を変えることはないため、先読みしても値は一貫している

## 17. ポスターフッター帯・ヘッダー縮小（2026-04-10 v10）

### Q: ウォーターマークが8枚目カードと重なっている

**決定: `FOOTER_H=36px` フッター帯を新設し、ウォーターマークをその中に配置**

- `grid_h = CANVAS_H - header_h - FOOTER_H` に変更。カード描画範囲の下端が `CANVAS_H - FOOTER_H` で確定するため重なりが構造的に解消される
- フッター帯はカード描画後に上書き描画（`theme["header"]` 背景 + `theme["accent"]` 4px ライン）
- ヘッダーと対称デザインにすることで、ポスター上下のバランスが整う
- ウォーターマークは `textbbox` で正確な文字高さを計測し、フッター帯内で縦中央揃えに配置

### Q: ヘッダー 120px は広すぎる

**決定: `HEADER_H: 120 → 96 px`（-20%）**

- 縮小分（24px）がカードへ回らない（FOOTER_H=36px 追加で相殺以上）
- カード高さ: 8スロット 235→232px (-3px)、10スロット 211→204px (-7px) と影響は最小限
- 64pt フォントのテキスト高さ（約85px）が 96-4=92px の有効領域に収まることを確認済み

## 18. 並び替えモード UX 改善（2026-04-10 v10）

### Q: 並び替えモード中も編集ボタンが反応してしまう

**決定: `render_slot_card(i, disabled=False)` に `disabled` パラメータを追加**

- `disabled=True` 時は `st.button(..., disabled=True)` が渡され、Streamlit の標準動作としてホバーエフェクトが抑制される（CSS 追加不要）
- 並び替えモードで `is_reorder = st.session_state["reorder_mode"]` を取得し `render_slot_card(slot_idx, disabled=is_reorder)` で渡す

### Q: 並び替えモードでカードグリッドが非表示になりゲーム情報が確認できない

**決定: カードグリッドを `if/else` から常時描画に変更**

- 並び替えモード時は `sort_items` リスト（上段）＋ カードグリッド（下段、編集ボタン無効）を両方表示
- ゲームのサムネイル・レビュー文を参照しながら順序を決められるため、並び替え品質が向上

## 19. テーマカラー変更・セール強調・ボタン整理（2026-04-10 v11）

### Q: UI が地味で視線誘導が弱い

**決定: `primaryColor` を Steam セールグリーン `#A4D007` に変更**

- Steam のセール・購入ボタンと同系色の鮮やかなグリーンを採用
- `type="primary"` のボタン（ポスターを生成）と `st.progress` が自動的にグリーンに変化
- ダークネイビー背景（`#1b2838`）との高コントラストにより視線誘導が強化される
- config.toml の変更のみで全 primary 要素が一括更新できる Streamlit のテーマ機能を活用

### Q: セール価格の割引率がポスター上で目立たない

**決定: `SALE_GREEN = (164, 208, 7)` を定義し割引率テキストを分割描画**

- `primaryColor` と同じ `#A4D007` を PIL RGB タプルとして定数化
- 割引価格フォーマット `"-XX%  ¥YYY"` は `"  "` (2スペース) を区切りとして生成されているため、`split("  ", 1)` で安全に分割できる
- `draw.textlength(disc_part + "  ")` でプレフィックス幅を計測し定価テキストの描画位置を決定
- バッジサイズ計算はフルテキストで行い、描画のみ分割する（サイズ計算の変更は不要）

### Q: ダウンロードボタンと生成ボタンが同じ緑で「重要度が同じ」に見える

**決定: `st.download_button` の `type="primary"` を削除**

- 「ポスターを生成」だけが primary（グリーン）= 最も重要なアクションという視覚的ヒエラルキーを確立
- ダウンロードは「生成後の補助アクション」として secondary 扱いが適切
- ダイアログの「保存して閉じる」は各コンテキストの主アクションのため primary を維持
