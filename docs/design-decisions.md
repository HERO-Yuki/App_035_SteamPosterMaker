# SteamPosterMaker — デザイン・仕様決定ログ

開発中に議論・決定した UI/UX 仕様をまとめた記録です。  
「なぜそうなったか」を後から追えるようにしています。

---

## 1. レイアウト

### Q: グリッドは横4×縦2 と 横2×縦4 のどちら？

**決定: 横2×縦4（1行に2タイトル、4段）**

| 比較 | 横4×縦2 | **横2×縦4（採用）** |
|------|---------|----------------|
| 1カードの横幅 | 狭い（約460px） | 広い（約950px） |
| タイトルの読みやすさ | 長いタイトルが詰まる | 余裕がある |
| サムネイル | 小さい | 大きく表示できる |

Twitter向け横長 1920×1080 に「縦4段」が最も見やすい。

---

### Q: 全体見出し ON/OFF で何本紹介できる？

**決定: ON → 8本（ヘッダー 120px + 2×4）／ OFF → 10本（2×5）**

- 画像サイズは常に 1920×1080 px で固定
- ヘッダーなしにすると 5段になり 10本紹介できる
- toggle `st.toggle("全体見出しを表示する")` で切り替え

---

### Q: カード間のマージンは？

**決定: 10px（当初 20px から変更）**

- 20px では余白が目立ち、カードが小さく見えた
- 10px に変更 → カードサイズが約 950×227 px（8本モード）に拡大

## 3. 背景スタイル

**決定: 2択（ぼかし / 単色）**

- **ぼかし**: ゲームのヘッダー画像を拡大・ガウスぼかし（強度スライダー 1〜40）
- **単色**: テーマカラーの背景色を使用
- 年齢制限ゲームのカードはぼかし処理をスキップ（錠前画像を使うため）

---

## 4. 価格バッジ

### Q: 価格の表示位置は？

**決定: サムネイル右下・半透明バッジ（サムネ端から 10px 余白）**

- 当初はテキストエリア内に表示していた
- サムネ右下バッジに変更 → テキストエリアの高さが増え、レビュー文が多く書ける
- バッジ: 黒半透明背景 + アクセントカラー文字

### Q: バッジのサイズ感は？

**決定: フォント 24pt・端から 10px の余白**

- 当初フォント 16pt、端ビタ付け
- 16pt は小さすぎて読みにくいと判断 → 1.5倍の 24pt に変更
- 「ビタ付け」が気になるとのフィードバック → `EDGE_MARGIN = 10px` を追加

---

## 5. 年齢制限コンテンツ

### Q: Steam API が年齢制限で詳細を返せない場合の表示は？

**決定: テキスト「エラー」ではなく、Pillow で描画した錠前アイコン + "18+"**

- カード背景: ダークレッド
- 錠前ボディ（角丸矩形）+ アーチ（楕円×2）+ キーホール（円+三角）を Pillow で描画
- UI プレビューカードには `🔞` 絵文字を表示
- `@lru_cache(maxsize=4)` で同サイズの錠前画像をキャッシュ

---

## 6. レビュー文

### Q: 文字数上限をどう扱う？

**決定: 入力は自由、140字超で保存不可（リアルタイムカウンター付き）**

| 案 | 内容 |
|----|------|
| `max_chars=140` で強制打ち切り | 絵文字を含む長い文が途中でカットされる違和感 |
| **リアルタイムカウンター + 保存無効化（採用）** | 何文字入力したかが見え、超過時に「減らしてから保存」と案内できる |

- 140字超: カウンター文字色が赤、`st.error` 表示、「💾 保存して閉じる」が `disabled=True`

### Q: ポスター上のフォントで実際に何文字収まる？

- 現在のフォントで等幅換算すると約 148文字相当が4行に収まる見込み
- ただしゲーム名・価格・プレイ人数との兼ね合いで **140文字を上限として維持**
- ポスター上では収まらない場合、フォントサイズを 1pt ずつ自動縮小

---

## 7. 全体見出し（ポスタータイトル）

### Q: 文字数上限は？

**決定: 25文字**

- ヘッダー幅 1920px、フォント 64pt、日本語全角 1文字 ≈ 70px
- 左右パディング 80px を引くと `(1920 - 80) / 70 ≈ 26.3` 文字
- 日本語タイトルを想定して安全マージンを取り **25文字** に設定
- UI 側は `max_chars=25` で強制

---

## 8. スロット並び替え（ドラッグ＆ドロップ）

### Q: ドラッグ＆ドロップが効かない（不具合）

**原因**: `sort_items()` を `st.expander` 内に置いていたため、ドラッグ後の rerun でエキスパンダーが閉じ、ユーザーに「変化なし」に見えた。

**決定: 並び替えモードのトグルを導入**

- スロットヘッダー行に **「🔀 並び替え」ボタン** を追加
- 押すと 2×N グリッドが**縦1列のソータブルリスト**に切り替わる
- 「✅ 並び替え完了」ボタンで通常グリッドに戻る
- ドラッグ領域が明確になり、誤操作も防止

---

## 9. インポート機能

### Q: 「最近プレイしたゲーム」「プレイ時間 TOP」を API 経由で自動取得できる？

**当初実装 → 削除**

- Steam Web API の `IPlayerService` は **API キーが必要**
- ユーザーへの API キー発行・入力の負担が大きいと判断
- **ウィッシュリスト取得（API キー不要）のみ残した**

| 機能 | 必要なもの | 採用 |
|------|-----------|------|
| ウィッシュリスト | なし（公開設定のみ） | ✅ 採用 |
| 最近プレイしたゲーム | Steam API キー | ❌ 削除 |
| プレイ時間 TOP | Steam API キー | ❌ 削除 |

## 10. UX 細部

| 事項 | 決定内容 |
|------|---------|
| API 呼び出し中のフリーズ | `st.spinner` / `st.status` で進捗を表示 |
| 生成完了演出 | `st.balloons()` → **`st.toast()`** に変更（風船が邪魔との意見）|

| ダウンロードボタン | 全幅（`use_container_width=True`）|
| スロットナンバリング | ポスター上のバッジ番号は**削除**（不要との判断）|
| サイドバー | 開発者フォローリンク（X @Yuki_HERO44）を追加 |
| 検索のトリガー | Enter キー送信 (`st.form`) に対応（ボタンクリック必須だったため）|

## 11. プレビューカード（UI）

| 事項 | 決定内容 |
|------|---------|
| レビュー文の表示位置 | 右カラムではなく**カード下全幅**に表示（読みやすさ優先）|
| 行間 | タイトル・価格・人数を1つの HTML ブロックに統合して `margin:0` で詰める |
| フォントサイズ | `0.78rem` → `0.88rem`（視認性改善）|
| 文字色 | `#aaa` → `#ccc`（やや明るく）|
| カード間 CSS gap | `2px`（Streamlit デフォルトの間隔を縮小）|

## 12. アプリ名変更・定数集約（2026-04-10 v5）

### Q: アプリ名は「Steam8 Poster」のままでよいか？

**決定: `SteamPosterMaker` に変更**

- "8" の数字が10本対応後に誤解を生む（8本専用アプリに見える）
- 動詞 "Maker" を付けることで「作るツール」の意図が伝わりやすい
- 変更箇所: `APP_NAME` 定数・`st.title`・`page_title`・ファイルコメント（1か所変えれば他は自動追従）

### Q: カード間マージンをどこまで狭められるか？

**決定: `MARGIN = 4 px`**

| 値 | カードサイズ（見出しON） | 印象 |
|----|------------------------|------|
| 20 px（初期） | 930×215 px | 余白が広く窮屈 |
| 10 px | 950×227 px | バランスが良い |
| **4 px（採用）** | **954×235 px** | 余白最小・コンテンツ最大化 |

4px は「隣のカードと完全に接しない最小の視覚的区切り」として残す。0にすると境界が消えて読みにくくなるため採用しない。

### Q: ハードコードされた数値が `compute_layout` や `draw_card` に散在していた

**決定: `app.py` 冒頭の定数セクションに集約**

- グリッド系: `HEADER_H`, `PLAYER_H`, `ROW_GAP`, `TITLE_MAX_H_ON/OFF`
- タイポグラフィ系: `HEADER_FONT_PT`, `TITLE_FONT_PT`, `TITLE_MIN_PT`, `PLAYER_FONT_PT`, `PLAYER_MIN_PT`, `REVIEW_FONT_PT`, `REVIEW_MIN_PT`, `PRICE_FONT_PT`, `SLOT_PH_FONT_PT`, `WM_FONT_PT`
- 既存: `MARGIN`, `THUMB_W`, `SEPARATOR_W`, `TEXT_PAD`, `PRICE_BADGE_PAD`, `PRICE_BADGE_EDGE`

これにより「どの数値を変えればどこに影響するか」がファイル冒頭だけで把握できる。
